# EP-961

## Proof Attempts and Literature Notes

### Working statement
Let `f(k)-1` be the maximum length of a block of consecutive `k`-smooth integers. Estimate growth of `f(k)`.

### Deep standalone computation (`ep961.mjs`)
- Method: exact largest-prime-factor sieve up to `X`, then simultaneous run-length tracking for many `k` values.
- Deep run: `X = 30,000,000` and
  `k in {10,15,20,25,30,40,50,60,80,100,120,150,200,300,500,700,1000,1500,2000,3000,4000,5000}`.

### Selected lower bounds from run witnesses
- `k=100`: max run `13`  -> `f(100) >= 14`
- `k=500`: max run `19`  -> `f(500) >= 20`
- `k=1000`: max run `33` -> `f(1000) >= 34`
- `k=2000`: max run `27` -> `f(2000) >= 28`
- `k=5000`: max run `35` -> `f(5000) >= 36`

### Interpretation
The deep scan strengthens finite lower bounds for many `k`, but does not provide an asymptotic proof for the true order of `f(k)`.

### Formal lemmas from definition
Let \(R(k)\) be the maximum run length of consecutive \(k\)-smooth integers; by definition \(f(k)=R(k)+1\).

1. Monotonicity:
\[
k_1\le k_2 \implies R(k_1)\le R(k_2)\quad\text{and}\quad f(k_1)\le f(k_2),
\]
because every \(k_1\)-smooth integer is also \(k_2\)-smooth.

2. Certified lower bounds:
if a concrete interval \([a,a+L-1]\) is verified to consist entirely of \(k\)-smooth numbers, then
\[
R(k)\ge L,\qquad f(k)\ge L+1.
\]
Hence each witness interval in `data/ep961.json` is a rigorous certificate of the corresponding lower bound.
