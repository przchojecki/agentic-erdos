# EP-600 partial

## Statement
Let `e(n,r)` be minimal so that every `n`-vertex graph with at least `e(n,r)`
edges, each edge in a triangle, has an edge contained in at least `r` triangles.
Ask about `e(n,r+1)-e(n,r)` and `e(n,r+1)/e(n,r)`.

## Attempt in this batch
I ran randomized local search for feasible graphs where each edge has triangle
count in `[1,tmax]` (`tmax=r-1`), producing constructive lower bounds
`e(n,r) >= m+1`.

Data file:
- `data/ep600_local_search_bounds.json`

## Result
Found explicit feasible edge counts (examples):
- `tmax=1` (`r=2`): at `n=30`, found `m=54` so `e(30,2) >= 55`.
- `tmax=2` (`r=3`): at `n=30`, found `m=145` so `e(30,3) >= 146`.

For one point (`n=26,tmax=1`) no feasible graph was found in this heuristic run.

## Hard point
These are constructive lower bounds from heuristic search, not exact values or
asymptotic proofs of the two limit questions.
