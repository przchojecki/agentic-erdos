# EP-611

## Problem Statement
If every maximal clique in an \(n\)-vertex graph has size at least \(cn\), must the clique-transversal number \(\tau(G)\) be \(o_c(n)\)?

## Literature / Context
- Known generic bound yields only linear control in this regime.
- The open part is whether linear-size maximal cliques force truly sublinear \(\tau(G)\).

## Our Approaches / What Is Proven
- Built exact finite profile computations:
  - maximal cliques via Bron-Kerbosch,
  - exact \(\tau(G)\) via branch-and-bound hitting set.
- Included deterministic high-maximal-clique families and filtered random samples.

## Computation-Guided Observations
- Deterministic families:
  - complete graph: \(\tau=1\),
  - complete minus perfect matching: min maximal clique size \(=n/2\), \(\tau=2\).
- Filtered random cases with small accepted \(c\)-thresholds also show small \(\tau/n\) in sampled windows.
- Finite evidence is compatible with small transversals when maximal cliques are large, but not asymptotic proof.

## Next Proof Targets
- Characterize structural conditions that force bounded (or slowly growing) transversal size from large maximal cliques.
- Seek extremal constructions maximizing \(\tau\) under min-maximal-clique \(\ge cn\) to test sharpness.
