## Deeper Proof Program (A => B => C)

### Target

### Step A: Clique-tree charging scheme
Represent a chordal graph by a clique tree and assign each edge to one bag while
controlling overlaps.

### Step B: Degree-constrained optimization
Translate maximal overlap constraints into a quadratic program on bag sizes and

### Step C: Extremal template identification

## LaTeX Proofs for A => B => C Skeleton

### Proposition M1 (Tree decomposition counting identity)
Let \(G\) be chordal with clique tree \(\mathcal T\), maximal cliques
\(\{K_i\}\), and separators \(S_e\) on edges \(e\in E(\mathcal T)\). Then
\[
|E(G)|=\sum_i \binom{|K_i|}{2}-\sum_{e\in E(\mathcal T)}\binom{|S_e|}{2}.
\]

\[
\textit{Proof.}
\]
Every graph edge appears in all bags along a connected subtree of
\(\mathcal T\). Inclusion-exclusion on tree edges subtracts repeated counts
exactly once per separator incidence, giving the formula.
\(\square\)

### Proposition M2 (Optimization reduction)
If one proves a sharp universal upper bound for
\[
\sum_i \binom{|K_i|}{2}-\sum_{e}\binom{|S_e|}{2}
\]
under chordal feasibility constraints, then the corresponding edge-clique
partition upper bound follows.

\[
\textit{Proof sketch.}
\]
Use M1 to convert structural constraints to a finite optimization problem; then
translate each extremal configuration back to a chordal graph template.
\(\square\)
