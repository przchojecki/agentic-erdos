# EP-369

## Proof Attempts and Literature Notes

## Definitive Status (Locked)

As written, EP-369 is definitively true.

- For fixed `epsilon > 0` and `k >= 2`, taking the fixed block
  `1,2,...,k` gives the result once `n > k^(1/epsilon)`.
- Therefore the literal statement is proved with an explicit threshold.

What remains nontrivial are only corrected variants (e.g. local
`m^epsilon`-smoothness, or requiring the block near `n`), which are separate
questions.

## Nontrivial Variant Split (Balog-Wooley Framing)

The literal statement is trivial. The meaningful variants are:

1. Local smoothness variant:
   each term in the block is required to be `m^epsilon`-smooth (term-wise).
   This is positive by Balog-Wooley (1998): for any `epsilon>0` and `k>=2`,
   infinitely many `m` have `m+1,...,m+k` all `m^epsilon`-smooth.
2. Near-top-window variant:
   require the block to lie near the top, e.g. inside `[n/2,n]`.
   Balog-Wooley gives positivity for infinitely many scales, but the
   “all sufficiently large `n`” form is open.

# EP-369 full solution (as written)

## Statement
For fixed `epsilon>0` and `k>=2`, ask whether for all sufficiently large `n`
there is a block of `k` consecutive integers in `{1,...,n}` all of which are
`n^epsilon`-smooth.

## Claim
The statement is true as written.

## Proof
Take the fixed consecutive block

`1,2,...,k`.

If `n > k^(1/epsilon)`, then `n^epsilon > k`.
For each `m in {1,...,k}`, every prime factor of `m` is at most `m<=k<n^epsilon`.
Hence each such `m` is `n^epsilon`-smooth.
Therefore, for all sufficiently large `n` (namely `n > k^(1/epsilon)`), the
block `{1,...,k}` is a sequence of `k` consecutive integers in `{1,...,n}` that
are all `n^epsilon`-smooth.

So the problem, as written, is proved.

## Note
The background indicates nontrivial corrected variants (e.g. requiring
`m^epsilon`-smoothness for each term, or requiring the block near `n`).


## Deep Standalone Computation (2026-03-06)

The literal statement is trivial (fixed block `1,\dots,k`), so computation targeted corrected variants.

  - `\varepsilon=0.25`: 3
  - `\varepsilon=0.3`: 4

- Variant B (near-top window with threshold `N^{\varepsilon}` at `N=2\cdot10^8`):

Proof-direction note: corrected variants show strong epsilon sensitivity; short runs for `\varepsilon\le 0.3` in large top windows suggest that proving long consecutive stretches may require significantly larger `\varepsilon` or weaker locality constraints.
