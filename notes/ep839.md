# EP-839 partial attempt

## Route
Built the lexicographically-greedy sequence `a_1 < a_2 < ...` avoiding values that are sums of consecutive earlier terms.

## Evidence from this batch
- `data/ep839_consecutive_sum_free_greedy.json` up to `n=3000`:
  - `a_n / n` grows from about `2.64` (`n=50`) to `3.575` (`n=3000`).
  - normalized reciprocal prefix `sum_{j<=n} 1/a_j / log(a_n)` drops from `0.646` to `0.489`.

## Hard point
Greedy behavior is not extremal for either density or reciprocal-sum asymptotics, so it does not settle whether `limsup a_n/n = infinity` must hold for all valid sequences.

## Status
Heuristic growth signal only; theorem-level conclusion remains open.
