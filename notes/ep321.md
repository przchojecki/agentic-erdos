# EP-321 partial

## Statement
Let `R(N)` be the largest size of `A subseteq {1,...,N}` such that all subset sums
`sum_{n in S} 1/n` are distinct. Estimate `R(N)`.

## Attempt in this batch
I ran exact branch-and-bound for small `N`, converting reciprocals to exact integer
weights by scaling with `lcm(1,...,N)`.

Data file:
- `data/ep321_exact_small_scan.json`

## Result
Exact values were computed for `8 <= N <= 25`.
- `R(25)=18` (exact).
- Ratio `R(N)/(N/log N)` in this range is about `1.82` to `2.33`.
- Runtime grows sharply (e.g. `N=25` took about 37.8s).

## Hard point
Finite exact data supports `R(N)` on `N/log N` scale but does not resolve the
iterated-log correction terms in the background bounds.
