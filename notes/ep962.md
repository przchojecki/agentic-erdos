# EP-962 deeper attempt with search

## Statement
`k(n)`: longest block `m+1,...,m+k` (for some `m<=n`) where each term has a prime
factor `>k`.

## Experiment in this attempt
I ran a prefix search to `n=10^6` using greatest-prime-factor tables.

Data file:
- `data/ep962_prefix_search_n1e6.json`

Observed:
- best `k` up to `10^6` is `121` (first found near `m=879498`).
- `k/sqrt(n)` decreases over sampled scales.

## Interpretation
Finite-range data suggests sub-polynomial growth and no near-`n^epsilon` behavior in
this range, but does not prove asymptotic bounds.

## Formal lemmas from the definition

Let \(K(n)\) denote the EP-962 quantity (called `k(n)` above).

### Lemma 1 (monotonicity in \(n\))
If \(n_1\le n_2\), then
\[
K(n_1)\le K(n_2).
\]

Proof:
Any witness block with start \(m\le n_1\) is also admissible for \(n_2\).

### Lemma 2 (universal upper bound \(K(n)\le n\))
For every \(n\ge 1\),
\[
K(n)\le n.
\]

Proof:
Suppose a witness of length \(k\) starts at \(m+1\) with \(m\le n\).
Each term in the block has a prime factor \(>k\), so in particular each term is \(>k\).
Hence the first term satisfies \(m+1>k\), i.e. \(m\ge k\).
Since \(m\le n\), we get \(k\le n\).

### Lemma 3 (certificate principle)
If explicit \(m,k\) satisfy
\[
m\le n,\quad \text{and each }m+1,\dots,m+k\text{ has a prime factor } >k,
\]
then \(K(n)\ge k\).

Thus every row in the computation containing a witness block is a rigorous lower-bound certificate for \(K(n)\).
