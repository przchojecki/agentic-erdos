## Deeper Proof Program (A => B => C)

### Target
Prove every \(n\)-vertex graph decomposes into \(O(n)\) cycles and edges, ideally
with best possible constant.

### Step A: Dense-core extraction
Find a dense subgraph where long cycles can be peeled efficiently.

### Step B: Expansion-based cycle packing
Use robust expansion to remove many edges per cycle while controlling leftovers.

### Step C: Sparse remainder handling
Bound residual edges/forest-like pieces by \(O(n)\), closing total count.

### Concrete Blocking Lemma (Most Critical)
Need a cycle-packing lemma that guarantees linear edge-removal efficiency in
near-extremal sparse-expansion regimes.

## LaTeX Proofs for A => B => C Skeleton

### Proposition BV1 (Decomposition accounting)
If one removes cycles \(C_1,\dots,C_t\) and leaves graph \(H\), then total number
of parts is \(t+|E(H)|\).

\[
\textit{Proof.}
\]
Each removed cycle contributes one part; leftover edges are singleton parts.
\(\square\)

### Proposition BV2 (Linear bound reduction)
If there is a procedure ensuring
\[
t\le c_1 n,\qquad |E(H)|\le c_2 n,
\]
for all \(n\)-vertex graphs, then total decomposition size is
\((c_1+c_2)n=O(n)\).

\[
\textit{Proof.}
\]
Apply BV1.
\(\square\)
